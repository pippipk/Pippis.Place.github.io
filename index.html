Relevant Implications

- We have future proofed the design of this program. Future Proofing means the program is designed so new features can be added easily and the old ones easily removed.

- We have been adding things this whole time, in each of the sprints. We addressed this in the way the programme has been designed.

- Program elements (colours, buttons and shapes) have all been programed as objects. This means each object is a self contained module piece of code that takes a defined set of parameters to work. Most take basic mouse data and colours to work. All objects are built to a similar structure and compulsorily contain an update function. This means they can appear on the interface using the update call. Management of most of the objects happens in a central place (Control Object). The idea is if you give the program to someone else (an experienced programmer), they could interpret what it does and built a new component. They could remove features and add new features.

- Comments in the code are precise and informative.
